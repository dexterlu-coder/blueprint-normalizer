# 决策备忘：标题栏位置多证据仲裁

日期：2026-06-28

## 当前决策

本项目采用“标题栏位置多证据仲裁”，不采用“无条件全量三方平权多数票”。

最终目标链路为：

1. PDF 图纸包拆分为一页一个 PDF。
2. 渲染单页图像。
3. 定位标题栏位置。
4. 根据标题栏正确方向应在下方或右下方的规则，推导页面已旋转角度和校正角度。
5. 旋正单页 PDF。
6. 在旋正后的标题栏区域 OCR。
7. 抽取图纸图号。
8. 用图号重命名单页 PDF。

默认目标是无人参与；只有证据不足、证据冲突、VLM 仍不确定、图号抽取或文件命名风险高时，才进入人工异常队列。

## 被废弃的设计

废弃：

- 无条件对每页都运行 OpenCV、MCP/VLM、人工或 OCR。
- 将各方结果视为同权票。
- 用简单多数票直接决定最终旋转角度或文件名。
- 让云端 VLM 作为默认主流程。

废弃原因：

- 多个证据源回答的不是同一个问题，不能同权投票。
- OCR 主要证明字段簇和图号，不擅长单独判断整体布局。
- VLM 有外发、版本漂移、置信度不可校准和成本问题。
- 人工参与应是异常处理，不应是常规步骤。
- 文件重命名属于高风险动作，需要可追溯证据而不是黑箱多数票。

## 保留并升级的设计

保留：

- 旧三方比对作为早期 ground truth 建立和算法误判定位方法。
- 多源证据互相校验的思想。
- OpenCV、YOLO/OBB、OCR、VLM、人工都可以参与，但角色不同。

升级为：

```text
标题栏位置多证据仲裁
```

证据角色如下：

| 证据源 | 当前角色 | 自动化地位 |
| --- | --- | --- |
| YOLO/OBB | 标题栏候选框、边侧位置、候选角度主证据 | 本地主线候选定位 |
| OpenCV | 图框、表格线、贴边、结构和边缘密度独立校验 | 本地几何校验 |
| OCR / RapidOCR | 标题栏字段簇真实性、文字方向、后续图号抽取 | crop 级字段证据 |
| VLM | 本地证据冲突、低置信、新版式时的疑难解释 | 可禁用兜底 |
| 人工 | 自动证据不足或命名风险高时最终复核 | 异常队列 |

## 自动接受条件草案

满足以下条件时，可进入自动接受候选：

- 仅有一个强标题栏候选，或多候选已被后处理明确仲裁。
- 候选贴图纸外框线，且不是图纸主体内部表格。
- YOLO/OBB 位置与 OpenCV 几何证据一致，或冲突可由已知规则解释。
- OCR 字段簇命中人员流程字段和图纸属性字段中的关键组合，或在 OCR 不可用时有足够强的结构代理证据。
- 标题栏位置能唯一映射到 0/90/180/270 度。
- 旋正后标题栏 crop 可保存，且图号候选满足命名规则、无重名冲突。

不满足时，不静默通过，应进入 VLM 或人工异常队列。

## 证据保存要求

每页应保存结构化仲裁记录，建议命名为 `ArbitrationRecord`，至少包含：

- 输入 PDF 页码、渲染图路径、输出单页 PDF 路径。
- 标题栏候选框、候选来源、候选位置和候选角度。
- YOLO/OBB 原始结果与后处理结果。
- OpenCV 图框/贴边/表格结构证据。
- OCR 原文、字段簇命中、OCR 方向、OCR crop 路径。
- VLM 请求与响应路径，仅在触发且获批时存在。
- 仲裁结论：标题栏位置、已旋转角度、校正角度、接受/复核状态和原因。
- 旋正后标题栏 crop、图号候选、最终文件名候选、重名/非法字符/低置信风险。

这些证据服务两个目的：

- 自动流程可追溯。
- 人工异常复核时只展示必要信息，机器字段放入 JSON 或日志，遵守 `AGENTS.md` 的低噪声审核入口规则。

## 下一步建议

下一步应围绕 `ArbitrationRecord` 做实现计划，而不是继续扩展平权投票：

1. 规划标题栏位置仲裁记录 schema。
2. 把已有 YOLO/OBB 后处理、OpenCV 结果和 RapidOCR 字段簇报告统一到只读汇总报告。
3. 在 hard-case 和 round3 重点样本上回放，验证是否能稳定给出标题栏位置、旋转角度、OCR 字段簇和图号候选。
4. 再设计单页 PDF 旋正与图号重命名的 dry-run，不直接覆盖原文件。

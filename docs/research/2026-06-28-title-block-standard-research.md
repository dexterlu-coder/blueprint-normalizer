# 标题栏规范调研笔记

日期：2026-06-28

## 一、学习目标

- 我们真正要解决的问题：在 YOLO/OBB 标题栏检测出现零件、局部表格或多候选混淆时，用机械制图标题栏规范中的字段组合和表格结构规则辅助判断候选真实性。
- 本轮不做的事情：不重新训练模型，不修改标签，不实现 OCR/VLM，不处理完整 PDF，不把任何单个文字命中当作标题栏充分条件。
- 最终沉淀物：资料索引、字段组合规则、表格结构规则、长期规则更新。

## 二、关键判断

用户这次补充的方向是对的：标题栏不是“长得像表格的区域”，而是同时满足位置、图框关系、字段组合和结构布局的一类工程制图对象。

本轮调研后，标题栏判断应分层：

1. 强规则：标题栏贴图纸外框线，且在正确方向下位于下方或右下方。
2. 中强证据：多个标题栏角色字段组合出现，例如设计/制图、校对、审核、批准、日期等签字审核字段，叠加材料、比例、图号/图样代号、图名/名称等图纸属性字段。
3. 结构证据：标题栏内部通常由不同宽高的单元格组成，签字栏、日期栏、图名/图号/材料/比例区的格子大小并不完全一致。
4. 反证：候选区域不贴外框线、位于图纸主体内部、只有少量相似词、只是均匀表格或零件线框时，应降权或进入人工复核。

## 四、对象模型

本项目后续应把标题栏候选拆成这些对象：

- `FrameContactEvidence`：候选外边是否贴图纸外框线，是否与外框线存在空隙。
- `PositionEvidence`：候选位于页面下、右下、左、上、右哪一侧，并映射到旋转角度。
- `RoleFieldCluster`：设计、制图、校对、工艺、标准化、审核、批准、日期等人员/流程字段是否成组出现。
- `DrawingPropertyCluster`：图名/名称、图号/图样代号、材料、比例、重量、表面积、单位等图纸属性字段是否成组出现。
- `TableStructureEvidence`：单元格是否大小不一，是否存在签字/日期小格与图名/图号/材料等大格组合。
- `NegativeTableEvidence`：明细栏、技术要求表、局部零件表格或图纸主体内部线框造成的相似表格反例。

## 五、字段组合规则

标题栏 OCR/VLM 兜底不应只看单词，而应看字段簇。

人员与流程字段簇：

- `设计` 或 `制图`
- `校对`
- `工艺`
- `标准` 或 `标准化`
- `审核`
- `批准`
- `日期`

图纸属性字段簇：

- `图名`、`名称` 或明显的零部件名称区域
- `图号`、`图样代号` 或代号区域
- `材料`
- `比例`
- `重量`
- `表面积`
- `单位` 或企业/部门名称区域

建议判定方式：

1. 同时命中人员与流程字段簇、图纸属性字段簇，可信度高于只命中一组。
2. 命中 `设计/制图 + 校对 + 审核/批准 + 日期` 这类审核签字链时，说明候选更像标题栏。
3. 命中 `材料 + 比例 + 图号/名称` 这类图纸属性链时，说明候选更像标题栏。
4. 只出现 `材料`、`日期`、`审核` 之一，不足以判定标题栏。
5. OCR 结果不稳定时，允许同义字段或缺字容错，但必须保留人工复核门。

## 六、表格结构规则

标题栏常见结构不是简单均匀表格。它通常由多种功能区拼接：

- 签字/日期区：多行小格，常按设计、校对、审核、批准等角色排列。
- 图纸属性区：图名、图号、材料、比例、重量、表面积等字段所在格子大小不一。
- 单位/项目区：可能占据较大的连续单元格。

因此，后处理或 OCR 兜底可以把这些结构作为辅助：

1. 候选内部存在横竖线网格，但单元格面积分布不完全均匀。
2. 候选内部同时存在多个小签字格和少量大信息格。
3. 候选的长宽比、位置和图框贴边关系与标题栏一致。
4. 候选只是均匀阵列、零件轮廓或图纸主体内部表格时，应降低可信度。

## 七、反例与质量门

容易混淆的对象：

- 明细栏：可能也是表格，且常靠近标题栏，但它表达零件序号、名称、数量、材料等清单，不等于标题栏主体。
- 技术要求表：可能包含文字和条目，但缺少签字审核链和图号/图名/比例等完整组合。
- 零件局部结构：可能由线框组成，看起来像表格，但通常不贴图纸外框线，也不会包含标题栏字段簇。
- 局部说明表：可能出现材料、日期等少数字段，但组合不完整。

质量门：

1. 自动候选不能只凭字段命中通过。
2. 自动候选不能只凭表格密集通过。
3. 不贴图框线的候选，即使有局部表格结构，也应降权或复核。
4. 多候选时，一页最多输出一个最终标题栏；其余候选应作为 rejected 或 needs_review 记录。
5. 对标题栏综合表格中相连的上方/旁侧区域，应允许人工判断其是否属于标题栏整体，不要机械地把所有相连表格都判为误检。

## 八、我们的工作协议

后续 OCR/VLM 或后处理升级时建议按此顺序判断：

1. 先看候选是否贴图框线。
2. 再看候选位置是否符合标题栏方向映射。
3. 再看候选形态是否像标题栏：长宽比、贴边、单元格大小不一。
4. 再看字段组合：人员流程字段簇和图纸属性字段簇是否同时出现。
5. 最后处理反例：明细栏、技术要求表、零件内部表格、多候选冲突。
6. 任一强规则冲突或证据不足时，进入人工复核，不静默通过。

## 九、下一轮验证

最值得验证的不是立刻重训，而是设计一个小型 OCR/后处理诊断实验：

- 对 round3 中多候选或历史零件误检样本裁剪候选框。
- 尝试 OCR 提取候选内文字，按字段簇统计命中。
- 同时计算候选贴图框线、位置、单元格结构不均匀度。
- 输出机器报告，不直接改最终判断。
- 让用户只复核证据摘要是否符合肉眼判断。

成功标准：规则能解释 `sample_001`、`sample_009`、`unclear90_001_from_sample_001` 等零件误检，也能避免误伤 `aug90_002_from_sample_010` 这类用户认为可接受的综合标题栏样本。

# JS2207 标题栏图号 OCR 四模型测试计划

日期：2026-07-02

## 背景

用户已确认 MVP PDF 旋正输出中第 22 页仍然方向错误，但当前先不处理方向问题。下一步希望使用：

```text
tools/pdf_rotation_mvp/output/JS2207-00-00升降平台/
```

中的单页 PDF 测试 OCR 效果，目标是提取图号。用户希望测试：

- `qwen3.7-plus`
- `qwen3.7-max`
- `qwen3.5-ocr`
- `qwen-VL-ocr`

根据前期调研，模型 ID 规范化为：

- `qwen3.7-plus`
- `qwen3.7-max-2026-06-08`
- `qwen3.5-ocr`

## 标题栏规范依据

提示词应结合标题栏规范和本项目规则：

- 标题栏通常位于正确方向下的下方、底边满宽区域或右下方。
- 标题栏字段需要组合判断，而不是单词命中。
- 强字段包括图号/图样代号、图名/名称、材料、比例、重量/表面积、单位。
- 流程字段包括设计、制图、校对、工艺、标准化、审核、批准、日期。
- 明细表、技术要求表、局部说明表和零件表格不是标题栏。
- 只出现单个字段或零件表格中的编号，不能当作图号。

## 目标

1. 使用 MVP 输出的 29 个单页 PDF 作为输入。
2. 渲染每页为 PNG。
3. 基于已旋正页面裁切标题栏候选区域。
4. 使用四个模型提取图号。
5. 输出机器报告与低噪声人工审核表。
6. 发布审核包到固定入口：
   - `local_data/review_inbox/current/js2207_title_block_drawing_number_ocr_review/`

## 输入策略

MVP 输出 PDF 多数已旋正，标题栏应位于底部或右下区域。因此本轮 OCR 输入优先采用底部标题栏候选 crop，而不是整页图片。

默认 crop 建议：

- 对横向图纸：底部约 30%。
- 对竖向图纸：底部约 35%。
- 保留右侧较大区域和底部满宽区域，不再预设标题栏一定在右下角。

第 22 页方向已知存在风险，本轮不修复，但在报告中标记为历史方向风险页。

## 输出字段

人工审核表只保留：

- `页码`
- `模型`
- `模型提取图号`
- `图号是否正确`
- `正确图号`
- `备注`

机器字段放入 JSON/CSV 报告：

- source PDF。
- crop 路径。
- 原始响应。
- JSON 解析状态。
- schema 状态。
- 候选列表。
- 置信度。
- review reasons。

## 模型参数

通用：

- `temperature=0`
- 不设置 `top_p`

视觉模型：

- `response_format={"type":"json_object"}`
- `enable_thinking=false`

OCR 模型：

- 不使用 system message。
- 指令放入 user message。
- 尝试 `response_format={"type":"json_object"}`；若模型不支持，需要脚本记录解析失败或后续做兼容 fallback。

## 风险

- OCR 模型可能不支持与视觉模型完全相同的结构化输出参数。
- `qwen-vl-ocr-latest` 是 latest 模型，有供应商漂移风险。
- 第 22 页方向错误可能导致标题栏 crop 不在底部，OCR 结果可能无效。
- 第 3 页不是图纸或无标题栏，应该允许输出空图号并标记复核。
- 明细表中的零件号、材料号可能被误当作图号。

## 验收标准

1. 29 页均生成 crop 或失败记录。
2. 四模型总请求数应为 116。
3. 审核表行数应为 116。
4. 固定入口包含 HTML、Excel、crop 图片副本和报告。
5. 人工审核表不暴露内部字段。
6. 不重命名 PDF，不修改 MVP 输出 PDF。

## 背景

用户已查看 `local_data/title_block_ocr_diagnostic/review_summary.html`，并给出人工复查结论。

关键修正：

- `round3_val / aug90_002_from_sample_010` 的额外候选是图纸中另外表格的一部分，不是标题栏。
- `round3_round2_test / aug90_002_from_sample_010` 的额外候选同样不是标题栏。
- 其他复查样本可接受。
- `aug90_007_from_sample_020` 和 `sample_040` 的识别框整体约逆时针 3 度，但大致没有问题，可容忍。

这修正了此前“额外候选可能属于标题栏综合表格”的假设。后续多候选仲裁必须把这类普通表格候选当作非标题栏表格误检，而不是可接受标题栏候选；该规则不依赖候选出现在页面哪个方向。

## 后续建议

下一步应规划“YOLO/OBB 多候选后处理仲裁升级”，重点是：

- 一页最终只允许一个标题栏候选。

- 小角度 OBB 偏差在可容忍范围内时，不触发失败。
- 仲裁应优先选择真实标题栏候选，并将其他候选记录为 rejected 或 needs_review。

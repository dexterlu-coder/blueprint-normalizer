# YOLO/OBB 首训预测错误分层与改进计划

## 背景

YOLO/OBB 第二轮首训已经完成，训练流程、数据加载、标签格式和预测生成链路均跑通。

首训复查入口已归档：

```text
local_data/review_inbox/archive/round2_prediction_review_20260627_reviewed/
```

人工复查结果：

- 复查样本：14 张 val/test 预测图。
- 可以接受：9 张。
- 不可接受：5 张。
- 关键规则：一份图纸只有一个标题栏。

本计划只处理预测错误分层与改进路线，不直接启动再训练或完整 PDF 批处理。

## 目标

1. 将 5 个不可接受样本按错误类型分层。
2. 明确哪些问题先用后处理约束拦截。
3. 明确哪些问题需要补充样本或复查标注规则。
4. 明确再次训练前必须满足的质量门。
5. 为后续 OpenCV、YOLO/OBB、OCR/VLM 融合提供可靠证据边界。

## 非目标

本阶段不做：

- 不直接扩大训练集。
- 不直接调大模型。
- 不直接启动再训练。
- 不进入完整 PDF 批处理。
- 不接入 OCR/VLM。
- 不把训练产物、图纸或预测图提交到 Git。

## 错误分层

### A. 多框误检

涉及：

- `val/sample_009`
- `test/unclear90_001_from_sample_001`

判断：

- 这是当前最需要优先拦截的问题。
- 机械图纸业务规则明确：一份图纸只有一个标题栏。
- 多框结果不能静默通过，必须进入后处理仲裁或人工复核。

优先改进：

1. 后处理层增加 `single_title_block` 约束。
2. 每页最多输出一个最终标题栏框。
3. 如果模型输出多个候选框，应按置信度、几何位置、标题栏区域先验和框尺寸进行仲裁。
4. 如果仲裁后仍冲突，标记为 `needs_review`，不得自动通过。

### B. 零件误检成标题栏

涉及：

- `test/sample_001`
- `test/unclear90_001_from_sample_001`

判断：

- 这是模型将机械结构中的矩形/线框区域误认为标题栏。
- 仅靠模型置信度可能不够，因为误检框也可能高置信。

优先改进：

1. 补充负例或难例样本：包含容易被误判的零件矩形、表格状结构、局部框线。
2. 后处理加入标题栏几何先验：标题栏应靠近页面边缘，且在旋转校正候选方向下应落到下方或右下方。
3. 对远离边缘或位于图纸主体中部的候选框降低优先级。
4. 若候选框位置与 OpenCV 标题栏候选区域冲突，应进入人工复核或低置信状态。

### C. 框范围过大或越界

涉及：

- `val/sample_020`
- `test/sample_010`

判断：

- 这类问题未必影响方向判断，但会影响后续裁剪、OCR 或图号/章号识别。
- 对“标题栏检测器”来说，框明显越界或过度扩张不能算稳定可用。

优先改进：

1. 复查对应训练标签是否存在边界过大倾向。
2. 统一标注规则：标题栏框应覆盖完整标题栏，但不应大幅覆盖图纸主体或超出图纸有效边界。
3. 后处理可将预测框裁剪到图纸有效边界内，但裁剪不能掩盖模型本身定位质量问题。
4. 再训练前生成边界质量检查报告，单独统计越界框和过大框。

### D. 标题栏未完全覆盖

涉及：

- `val/sample_009`

判断：

- 这会影响后续图号、章号、标题栏 OCR/VLM 裁剪。
- 不能只看“命中标题栏”，还要看是否完整覆盖目标区域。

优先改进：

1. 复查训练集中类似标题栏的标注边界是否偏小。
2. 在 overlay 审核规则中加入“完整覆盖标题栏”作为单独判定项。
3. 预测复查时记录 `partial_cover`，与误检、越界区分。

## 改进优先级

第一优先级：后处理安全约束。

- 一页最多一个最终标题栏。
- 多候选必须仲裁或进入人工复核。
- 中部零件候选不得直接通过。
- 明显越界框不得直接通过。

第二优先级：数据与标注复查。

- 针对 5 个失败样本回看原图、标签和预测图。
- 判断失败来自训练数据不足、标注边界不一致，还是模型泛化不足。
- 必要时生成小批量难例补标任务，但仍需走固定审核入口。

第三优先级：再训练。

- 只有在后处理规则和标注复查计划明确后，才允许设计再训练。
- 再训练不能只追求 mAP，应继续使用 val/test 预测图人工复查作为质量门。

## 后处理候选规则草案

后续实现可考虑为每个预测框计算：

- `model_confidence`：模型置信度。
- `edge_proximity`：候选框到页面边缘的距离。
- `center_region_penalty`：候选框是否落在图纸主体中部。
- `size_reasonableness`：框面积和长宽比是否接近标题栏。
- `opencv_region_agreement`：是否与 OpenCV 标题栏候选区域一致。
- `rotation_side_consistency`：候选位置是否能映射到合理旋转方向。

最终输出不应直接等于模型原始框，而应分为：

- `accepted`：单一候选，满足几何与位置约束。
- `needs_review`：多候选、候选冲突、疑似零件误检或边界异常。
- `rejected`：候选明显不在标题栏合理区域。

## 下一步执行计划

1. 读取归档预测复查表和对应预测图。
2. 生成机器可读错误清单，保存在本地 ignored 目录。
3. 设计 YOLO/OBB 预测后处理脚本计划。
4. 设计失败样本复查包：原图、标签 overlay、预测 overlay 靠近展示。
5. 通过固定审核入口让用户确认错误原因归类。
6. 再决定是否补标、修标签或进行第二次训练。

## 质量门

进入下一轮训练前必须满足：

1. 5 个失败样本已有明确错误归因。
2. 多框输出已有明确后处理策略。
3. 零件误检样本已有负例/难例处理策略。
4. 框范围过大、越界和未完整覆盖已分开记录。
5. 固定审核入口中没有未归档任务。
6. RPD/TODO 已记录计划和回滚点。

进入完整 PDF 批处理前必须满足：

1. val/test 人工复查通过率达到稳定阈值。
2. 多框误检不会自动通过。
3. 错误样本能被分层输出，而不是被系统隐藏。
4. YOLO/OBB 只作为证据分支时，其失败不会破坏 OpenCV 主流程。

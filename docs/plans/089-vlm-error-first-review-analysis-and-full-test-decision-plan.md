## 背景

用户已完成 `local_data/review_inbox/current/ykj125_vlm_title_block_error_first_review/vlm_error_first_review.xlsx` 人工审核。用户希望总结哪些方案组合最好，并认为下一步可以根据最佳方案做全量测试。

本轮需要先读取人工填写结果，按“旋转角度是否正确”统计各模型/参数组合表现，再给出是否进入全量测试和推荐组合。全量测试仍需在总结后另按流程规划和执行。

## 目标

1. 读取当前固定审核入口中的已填写 Excel。
2. 统计各模型组合：
   - 人工正确数。
   - 人工错误数。
   - API/结果缺失数。
   - 备注中的风险信号。
3. 对照机器摘要，判断哪个方案最适合下一轮全量测试。
4. 形成清晰结论：
   - 推荐主方案。
   - 可选对照方案。
   - 暂不推荐方案及原因。
5. 记录分析结果到 RPD/TODO。

## 非目标

- 不直接调用模型跑全量。
- 不修改 prompt/schema。
- 不生成正式旋正 PDF。
- 不重命名 PDF。
- 不覆盖人工填写的 Excel。
- 不把本轮结果直接写入 ground truth。

## 实现计划

1. 新增或使用脚本读取 xlsx 底层 XML，不安装依赖。
2. 校验 Excel 行数为 60 且已填写。
3. 生成本地机器摘要到 `local_data/vlm_title_block_error_first_ykj125/`。
4. 更新 RPD/TODO。
5. 提交分析脚本和公开记录；不提交 `local_data`。

## 决策原则

优先级从高到低：

1. 人工审核旋转角度正确率。
2. API 成功率和 schema 稳定性。
3. 延迟和错误风险。
4. 是否为适合长期使用的稳定模型 ID。
5. 成本和全量审核负担。

## 验收标准

1. 输出每个模型组合的人工审核统计。
2. 明确推荐是否进入全量测试。
3. 明确推荐全量测试模型组合。
4. 当前固定审核入口不被覆盖。
5. 全量测试未在本轮直接执行。

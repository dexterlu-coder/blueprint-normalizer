# 细 ROI 审核入口说明修复计划

## 背景

用户打开细 ROI 审核入口后发现：

- `review_form.csv` 中各字段不知道应该填写什么。
- `review_index.html` 中不同颜色框没有说明。

该问题成立。当前审核入口虽然遵守了低噪声字段规则，但缺少必要的填写说明和颜色图例，影响人工审核效率。

## 目标

1. 在当前审核入口中补充 CSV 字段填写说明。
2. 在 `review_index.html` 中补充颜色框图例。
3. 在 `README.md` 中补充简明填写规则。
4. 保持人工表低噪声，不增加 bbox、score、长路径、JSON、候选列表或完整 OCR 字段。
5. 不改变细 ROI 生成算法和 OCR 实验结果。

## 非目标

本轮不做：

- 不重新设计细 ROI 算法。
- 不重新跑 OCR。
- 不修改机器候选结论。
- 不生成正式 PDF。
- 不重命名单页 PDF。
- 不归档当前审核入口。

## 字段填写说明

`review_form.csv` 字段含义：

| 字段 | 填写方式 |
| --- | --- |
| `序号` | 不填 |
| `样本编号` | 不填 |
| `细ROI判断` | 填 `正确`、`范围太大`、`范围太小`、`位置错误`、`看不清` 中最贴近的一项 |
| `图号判断` | 填 `正确`、`错误`、`未识别`、`不确定` |
| `人工确认图号` | 当图号判断不是 `正确` 时，填写人工看到的真实图号；若看不清可留空 |
| `备注` | 简短说明问题，例如裁掉右侧、仍包含图纸主体、字太浅、粗细候选冲突等 |

## 颜色图例

位置示意图中的框：

- 红框：右侧区域候选。
- 蓝框：右下区域候选。
- 绿框：表格线右侧区域候选。
- 较粗的框：当前机器选中的细 ROI。

## 验收标准

1. `review_index.html` 能看到颜色图例和填写提示。
2. `README.md` 能看到字段填写规则。
3. `review_form.csv` 字段不增加内部技术字段。
4. 人工 HTML/CSV 仍不暴露 bbox、score、长路径、JSON、完整 OCR 或候选列表。
5. `modified_pdf=false`。
6. `renamed_pdf=false`。

## 回滚准备

本计划、RPD 和 TODO 提交后作为说明修复前回滚点。若说明过多或不适合人工审核，可回退到该提交，重新调整文案。

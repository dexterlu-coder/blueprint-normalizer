# 标题栏位置多证据仲裁调研计划

## 背景

用户重新明确最终目标：

1. 先定位机械图纸标题栏位置。
2. 根据机械制图规则，标题栏正确方向应在下方或右下方。
3. 由标题栏当前位置推导页面需要旋转多少度。
4. 将图纸旋正。
5. 读取旋正后标题栏中的图纸图号。
6. 用图纸图号重命名单页 PDF。
7. 默认无人参与，只有证据不足、冲突或命名风险高时才进入人工判断。

此前文档中存在“三方比对”“条件并行兜底”“证据融合”“投票”等说法。需要整理这些设计的当前状态，避免后续把已过时的“平权多数票”误当主方案，同时保留历史设计以便追溯。

## 本轮目标

1. 记录当前设计结论：
   - 废弃：无条件全量三方平权多数票。
   - 保留并升级：面向标题栏位置的多证据仲裁。
2. 保留历史三方比对设计，不删除旧文档，只标注设计状态。
3. 调研外部类似案例和技术依据：
   - 工程图纸/技术图纸标题栏检测。
   - 文档图像方向检测。
   - 版面分析中的 region detection。
   - OCR 与目标检测结合抽取字段。
   - 多源证据融合而非简单投票的工业实践。
4. 沉淀新的设计备忘：
   - 标题栏位置仲裁对象模型。
   - 自动接受条件。
   - VLM/人工触发条件。
   - 证据保存要求。
   - 与最终 PDF 拆分、旋正、图号命名目标的关系。

## 非目标

本轮不做：

- 不改主流程代码。
- 不训练模型。
- 不调用 VLM。
- 不重新 OCR 图纸。
- 不处理完整 PDF。
- 不生成新的人工审核入口。
- 不删除旧设计文档。

## 调研问题

1. 机械图纸标题栏定位作为方向校正主依据是否有规范或案例支撑。
2. YOLO/OBB、OpenCV、OCR、VLM 各自适合在标题栏位置判断中承担什么角色。
3. 多源证据融合是否优于单一 VLM 或同权投票。
4. 严格数据安全约束下，本地 YOLO/OBB + OpenCV + OCR 是否是合理主线。
5. 标题栏位置、OCR 结果和图号候选应该如何作为中间资产保存。

## 判断标准

- 若外部案例支持“先检测标题栏/标题块，再抽取标题栏字段”，则当前主线成立。
- 若外部案例支持目标检测 + OCR 的组合，则 YOLO/OBB + OCR 路线成立。
- 若文档方向检测案例普遍依赖文字方向或版面方向，则 OCR/VLM 可作为辅助，但不能替代标题栏位置规则。
- 若外部方案没有机械图纸专用案例，也要记录差异，避免过度迁移。

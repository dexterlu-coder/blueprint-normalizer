# 竖向图纸底边标题栏规则固化计划

日期：2026-07-02

## 背景

用户补充说明：部分图纸是按纸张竖向绘制，标题栏位于下方并刚好占满图纸宽度，因此在 JS2207 Excel 审核中多处填写为“下方”。

这说明“下方”不是模糊描述，而是符合竖向纸张图纸的合法标题栏布局。后续不应强制把这类标题栏归入 `bottom_left` 或 `bottom_right` 四角类别。

## 目标

1. 将竖向纸张底边满宽标题栏写入长期机械图纸旋转规则。
2. 明确标题栏位置 schema 后续需要支持边位置，例如 `bottom_edge`、`top_edge`、`left_edge`、`right_edge`。
3. 保留“先判断标题栏当前位置，再确定性推导旋转角度”的流程。
4. 在 RPD 中记录本条用户补充对 JS2207 审核结果解释和后续 schema 设计的影响。

## 非目标

- 不重新调用 VLM。
- 不重跑 JS2207 审核归档脚本。
- 不修改已归档的 Excel 或本地审核结果。
- 不生成正式旋正 PDF。
- 不重命名 PDF。
- 不改动当前模型 prompt 或 schema 代码。

## 输出

- 更新 `reports/rpd-rotation-detection.md`。

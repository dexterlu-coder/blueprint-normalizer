# VLM 错题集旋转分组放宽评估计划

日期：2026-07-02

## 背景

用户指出：当前任务的实际目标是判断图纸旋转角度。标题栏位于上下左右哪个方向是重点，具体偏上或偏下、角位置或边位置不应作为主门限。

该判断成立。上一轮错题集小测用“精确标题栏位置”对照，会把 `right_edge` 与 `top_right` 计为不一致；但二者都代表标题栏位于当前图片右侧方向，对应图纸当前顺时针旋转 270 度。若项目目标是旋正图纸，则应以旋转分组为主评估口径。

## 目标

1. 保留精确标题栏位置字段作为诊断信息。
2. 新增旋转分组评估口径，按本地确定性映射计算：
   - `bottom_edge`、`bottom_right` -> 0 度。
   - `left_edge`、`bottom_left` -> 90 度。
   - `top_edge`、`top_left` -> 180 度。
   - `right_edge`、`top_right` -> 270 度。
3. 用已有错题集响应重算模型效果，不重新调用 API。
4. 重建当前审核包，使人工审核重点转为“派生当前旋转角度是否正确”。
5. 用户审核前仍不进入全量测试。

## 非目标

- 不重新调用模型。
- 不修改 VLM prompt/schema。
- 不生成正式旋正 PDF。
- 不重命名 PDF。
- 不把错题集结果写入 ground truth。
- 不把 OCR 模型混入本轮评估。

## 实现计划

1. 扩展 `scripts/experiments/build_vlm_title_block_error_first_review.py`：
   - 从上一轮人工正确标题栏位置派生人工正确旋转角度。
   - 从模型输出标题栏位置派生模型当前旋转角度。
   - 在机器摘要中同时记录精确位置命中率和旋转分组命中率。
2. 重建审核包：
   - Excel 展示模型标题栏位置、模型派生当前旋转角度、上一轮人工正确旋转角度。
   - 人工填写字段改为 `旋转角度是否正确`、`正确旋转角度`、`备注`。
   - 精确位置字段保留为参考信息，不作为人工主判断。
3. 更新 RPD/TODO：
   - 记录放宽口径的理由。
   - 记录使用已有响应重算的结果。
4. 验证：
   - `current` 固定入口仍只保留本轮待审核文件。
   - 图片副本仍在 `current` 内。
   - Excel 行数仍为 60，图片数仍为 15。

## 验收标准

1. 机器报告同时包含精确位置命中和旋转分组命中。
2. 审核 Excel 不再要求用户按角位置/边位置做主判断。
3. HTML 中明确展示模型派生当前旋转角度。
4. 本轮不联网、不生成正式 PDF、不重命名 PDF。
5. 用户审核完成前不扩大到全量图纸。

## 回滚准备

先提交本计划、RPD 和 TODO。若后续脚本调整或审核包重建不符合要求，可回退到该文档回滚点，保留“旋转分组优先”的需求记录。

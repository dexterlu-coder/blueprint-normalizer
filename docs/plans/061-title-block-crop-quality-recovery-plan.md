# 标题栏 crop 质量修复与命名审核重启计划

## 背景

63 条图号命名人工审核包已生成校正后页面和校正后标题栏 crop。用户只审核了部分样本，但反馈已经足够说明当前审核包不能继续作为最终命名依据：

- 所有已看样本的旋转方向均正确。
- 多个标题栏 crop 缺失右侧图名/图号区域。
- 少数 crop 基本完整，但图像浅、短横线不清，OCR 无法稳定识别图号。
- 当前 `review_form.csv` 已包含用户备注，应保留并作为缺陷样本输入。

因此当前命名审核应暂停，不要求用户继续填完 63 条。后续先修复标题栏 crop 质量门，再重新生成命名审核包。

## 已确认样本分层

### crop 覆盖错误或缺失

- `sample_008`：只截取标题栏左半部分，图名/图号在右侧未进入 crop。
- `sample_009`：crop 混入图纸主体和标题栏左侧，真实右下角图号栏未进入 crop。
- `sample_022`：只截取标题栏左半部分，图号区域缺失。
- `sample_032`：只截取标题栏左半部分，图号区域缺失。

### crop 基本正确但右侧边界不足

- `sample_016`：标题栏主体基本完整，但右侧少截，导致图号尾部缺失。
- `sample_006`：抽样发现右侧也存在截断风险，机器候选疑似少尾部字符。

### crop 基本完整但图像质量不足

- `sample_035`：标题栏完整，图号区域浅，短横线容易被 OCR 漏读或读成空格。
- `sample_039`：同类浅字、短横线不清问题。
- `sample_042`：标题栏基本完整，但图像很浅且有叠影感，需要增强和人工兜底。

## 根因判断

当前标题栏 OCR dry-run 的 crop 来源是 stage1 `best_candidate.bbox` 加固定 `CROP_PADDING_RATIO=0.03`。该策略存在两个硬伤：

1. 若候选框只覆盖标题栏左侧签字区，后续 crop、OCR、图号抽取都会继承错误。
2. 固定比例 padding 不能保证右侧图号栏完整，特别是横向长标题栏和右边界贴近页面边缘的图纸。

本问题不是单纯命名规则问题，也不能通过继续人工填写当前表格解决。

## 目标

1. 暂停当前 63 条命名审核，不要求用户继续填写。
2. 归档当前用户已填 `review_form.csv`，保留备注和人工判断。
3. 生成标题栏完整性专项审核入口，优先让用户确认 crop 是否完整，而不是确认图号。
4. 修复 crop 生成策略，使完整标题栏、图名和图号栏进入 OCR 输入图。
5. 对图像浅、短横线不清样本执行 OCR 图像预处理小实验。
6. 重新生成 63 条图号命名审核包，所有样本重新进入人工复核，不沿用旧的自动放行结论。

## 非目标

- 不生成正式旋正 PDF。
- 不重命名 PDF。
- 不把当前部分填写结果当作最终命名结果导入。
- 不跳过标题栏完整性人工复核。
- 不把内部分数、候选列表或调试 JSON 放入人工填写表。

## 工作阶段

### 阶段 1：保全当前审核反馈

1. 将当前 `local_data/review_inbox/current/naming_review/review_form.csv` 复制归档到业务输出目录。
2. 同时保存当前 `review_index.html`、`review_manifest.json`、`README.md` 和校正后图片索引。
3. 解析 CSV 时兼容 UTF-8-BOM 和 GBK/ANSI，避免 Excel 保存后乱码导致结果丢失。
4. 输出人工反馈摘要：
   - 已填写行数。
   - 旋转正确样本数。
   - crop 缺失样本。
   - 图像质量问题样本。
   - 需人工兜底样本。

### 阶段 2：标题栏完整性专项审核包

固定入口仍使用：

```text
local_data/review_inbox/current/
```

入口内容：

- `README.md`
- `title_block_crop_review/review_index.html`
- `title_block_crop_review/review_form.csv`
- `title_block_crop_review/assets/pages_corrected/`
- `title_block_crop_review/assets/crops_current/`
- `title_block_crop_review/assets/overlays/`

人工 CSV 只保留低噪声字段：

- `序号`
- `样本编号`
- `当前crop判断`
- `问题类型`
- `备注`

建议填写值：

- `完整`
- `右侧缺失`
- `左侧缺失`
- `错框`
- `混入主体`
- `字迹差`
- `不确定`

HTML 审核页应并排展示：

- 校正后整页。
- 当前 crop。
- 简洁的框选 overlay。

不得在 HTML 或 CSV 中展示内部状态码、分数、阻断原因、候选 JSON 或长路径。

### 阶段 3：修复 crop 生成策略

优先实现保守可靠策略：

1. 对已知正确旋转后的页面生成校正坐标系视图。
2. 对标题栏候选不再只使用 stage1 bbox。
3. 对底部/右下标题栏，按图框边界和标题栏位置扩展到完整标题栏区域。
4. 对右侧边界缺失的 crop，强制向图框右边界或页面右侧安全边界扩展。
5. 对左半标题栏误检，尝试合并同一底部带内的相邻表格区域，覆盖图名和图号栏。
6. 对无法自动确认完整性的样本，进入人工复核，不自动放行。

第一版可先生成候选 crop，不直接替换正式 OCR 输入，待人工审核确认后再进入 OCR。

### 阶段 4：图像预处理小实验

针对 `sample_035`、`sample_039`、`sample_042` 等浅字样本，复用既有 OCR 图像预处理调研结论做小实验：

- 灰度化。
- 自动对比度或 CLAHE。
- 锐化。
- 二值化或自适应阈值。
- 放大 2x/3x。
- 去噪。

每种增强输出图片、OCR 文本和图号候选，由人工可视化审核决定是否纳入正式流程。

### 阶段 5：重建命名审核包

在 crop 完整性和图像预处理策略通过人工审核后，重新生成 63 条图号命名审核包：

- 所有样本都重新进入人工审核。
- 不沿用旧包中的自动放行结论。
- CSV 保留低噪声字段。
- HTML 默认展示校正后页面、修复后标题栏 crop、必要时展示增强前后对比。
- 仍不生成或重命名 PDF。

## 验收标准

1. 当前部分填写结果已归档，且可读。
2. 固定入口只包含当前待审核任务文件。
3. 标题栏完整性审核包包含 63 条。
4. 人工表不暴露内部状态、长路径、调试分数或候选 JSON。
5. 对已知问题样本：
   - `sample_008`、`sample_022`、`sample_032` 不再只截左半标题栏。
   - `sample_009` 不再混入主体而遗漏右下图号栏。
   - `sample_016`、`sample_006` 右侧图号尾部不再截断。
   - `sample_035`、`sample_039`、`sample_042` 有增强前后对比和 OCR 结果。
6. 未生正式 PDF，未重命名 PDF。
7. 重新生成命名审核包前，必须先有人审核标题栏完整性。

## 回滚准备

每个阶段实现前先提交计划/RPD/TODO 回滚点。生成或替换 `local_data/review_inbox/current/` 前，先归档当前入口，确保用户已填备注不会丢失。

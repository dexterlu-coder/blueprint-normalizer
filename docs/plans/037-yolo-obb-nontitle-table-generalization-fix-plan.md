# YOLO/OBB 非标题栏表格误检泛化修正计划

## 背景

用户指出前一轮理解有偏差：问题不是“拒绝顶部蓝框”，而是拒绝把图纸中的普通表格错认成标题栏。由于图纸可能发生 0、90、180、270 度旋转，普通表格误检也可能出现在上、下、左、右任意方向。

当前文档和脚本中多处使用了“顶部蓝框”作为表述，容易让后续规则过拟合到方向或位置，这是需要立即修正的。

## 目标

1. 将规则表述从“顶部蓝框”修正为“任意方向的非标题栏表格误检”。
2. 修正 round3 多候选仲裁计划和 RPD 中的描述，避免方向特化。
3. 修正脚本中的回归命名，使 `aug90_002_from_sample_010` 只是当前代表样本，不是唯一方向规则。
4. 重新运行回归，确认输出结果不变，但语义更准确。

## 非目标

本阶段不做：

- 不重训。
- 不改标签。
- 不改变当前候选选择结果。
- 不引入 OCR/VLM。
- 不处理完整 PDF。

## 修正原则

- 标题栏判定应与图纸旋转角度解耦。
- 普通表格误检可能位于任意页面方向。
- 后处理不应写死“顶部”“上方”“蓝框”作为业务规则。
- `aug90_002_from_sample_010` 只是“非标题栏表格误检”的当前回归样本。
